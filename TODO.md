## 3. Supabase / マッチング深化

- ⭐ **本人確認付きRLS** — 現状はカジュアル用に anon 全許可。
  - 📈 device_id を匿名サインイン（`auth.signInAnonymously`）へ移行し、
    `auth.uid()` ベースのRLSで「自分の行だけ更新可」に締める。
  - 💻 `supabase.ts` の `deviceId()` を匿名認証セッションに置換。

- ⭐ **ゴースト鮮度管理** — 古い/低品質ゴーストの整理。
  - 💻 `find_opponent` を直近N日・同ラウンド帯フィルタ付きに。pg_cron で
    `ghost_builds` の古い行を間引く定期ジョブ。

- ◯ **不正対策** — クライアント計算のため盤やレートの改竄余地あり。
  - 💻 戦闘判定を Edge Function 化（`resolveBoard`/`simulate` をDenoへ移植し共有）。
    レート更新はサーバ側でのみ確定。`battle.ts` をプラットフォーム非依存に保つ。

- ◯ **週間/シーズンのリーダーボード** — `matches` 集計ビュー＋マテビュー。

- ◇ **リプレイ共有** — seed と両盤を保存し、URLで戦闘を再生（決定論シミュの強み）。
