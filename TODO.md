# 長期ロードマップ（フェーズ別 — タスク・ブレイクダウン）

四半期〜年単位の投資。フェーズは前から順に依存（前提が満たされて次が映える）。
各フェーズ冒頭に【ゴール】【完了の定義(DoD)】、続いて粒度を落とした実装タスク。

## Phase A — 「対戦ゲームとして成立」（MVP+）

【ゴール】ランクが信頼でき、端末を跨いで遊べる。
【DoD】レート確定はサーバのみ／匿名→正式の引き継ぎ動作／マッチが安定して成立。

- ⭐ **サーバ権威の戦闘判定**
  - [ ] `battle.ts` / `data.ts` / `rng.ts` を `packages/engine` 相当へ切り出し（Web/Edge共有）
  - [ ] Edge Function `resolve-match`：入力(両盤・seed・mode)→結果＋レート差分
  - [ ] クライアントは演出専用に、確定値はEdge応答で上書き
  - [ ] レート更新を `rank.ts` のロジックでサーバ側計算し `players` を更新
  - [ ] ✅ test: 黄金値テスト（同入力でクライアント `simulate` とEdge結果が一致）
  - [ ] レート確定APIにレート制限・冪等キー（二重加算防止）
- ⭐ **匿名→正式アカウント昇格**
  - [ ] `signInAnonymously` で開始、`linkIdentity` でメール/OAuth紐付け
  - [ ] `players.device_id` → `auth.uid()` 移行マイグレーション（既存行の引き継ぎ）
  - [ ] 名前変更・データ移行のUI（ホームのプロフィール欄）
  - [ ] ✅ test: 匿名で作った戦績が昇格後も保持される
- ◯ **マッチング品質**
  - [ ] `find_opponent` を「同帯→近傍→全体」の3段フォールバック化
  - [ ] 直近対戦相手の重複回避（最近K件を除外）
  - [ ] ゴースト不足時のPvE影フォールバック品質をラウンド帯で調整
- ◯ **シーズン & ソフトリセット**
  - [ ] `seasons` テーブル（開始/終了/報酬テーブル）
  - [ ] シーズン末にレート圧縮（例: 1500超を 1500 + (r-1500)*0.5）
  - [ ] 到達ティアに応じた報酬付与（コスメ/通貨）

## Phase B — 「同期リアルタイム対戦」

【ゴール】「相手が今いる」体験。準備同時進行→同時戦闘。
【DoD】2人以上が同一ロビーで準備し、同時に戦闘が始まり結果が一致する。

- ◇ **リアルタイム・ロビー基盤**
  - [ ] Supabase Realtime Presence でロビー参加/レディ状態を同期
  - [ ] `lobby.ts` にロビー状態機械（待機→準備→戦闘→精算→次R）
  - [ ] 準備フェーズのタイマー（全員レディ or 時間切れで進行）
  - [ ] 切断/再接続ハンドリング（離脱者はゴースト化して継続）
- ◇ **同期戦闘**
  - [ ] 盤(=入力)のみ Broadcast し、共有seedで両者が同一 `simulate` を再生
  - [ ] ✅ test: 2クライアントに同seed・同盤を渡し frame 列が一致
  - [ ] 結果はEdge（Phase A）で権威確定し、表示はローカル再生
- ◇ **8人ループ式トーナメント**
  - [ ] 毎ラウンド別の生存者と総当たりで対戦（ライフ制と相性◎）

- ◇ **観戦 & 実況リプレイ**

## 横断的な原則（どのフェーズでも守る）

- 🎮 まず「面白さ」。機能より1試合の手触りを優先。
- ⚖️ 数値は必ず `data.ts`/設定ファイルに集約し、コードに散らさない。
- 💻 戦闘・データのコアは純粋関数のまま（Web/Edge/ネイティブ共有の生命線）。
- 👤 スマホ縦持ちファースト。新機能はまず親指操作で検証。
- 📈 出した機能はテレメトリで効果測定し、5人会議で継続/撤退を判断。
- 🔒 ランクに関わる確定処理は必ずサーバ権威。クライアントは演出のみ。
- 🎨 新素材は GPT生成→ドット絵化パイプライン（冪等・再開可能）で量産。
- ✅ ロジック追加には必ず vitest。バランスは勝率レンジtestで自動ガード。
